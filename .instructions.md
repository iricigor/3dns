# 3DNS Project Instructions for GitHub Copilot

## Project Overview

**3DNS** is a web application for visualizing Continuous Glucose Monitoring (CGM) data from Nightscout. The project combines immediate 24-hour glucose insights with a roadmap to interactive 3D visualization.

### Current Focus
Build a modern, accessible 24-hour CGM dashboard that displays glucose trends, summary metrics, and time-in-range analysis. Future phases will add enhanced 2D views and eventually interactive 3D visualization.

## Technology Stack

### Current Implementation
- **Language**: Vanilla JavaScript (ES6+)
- **Markup**: Semantic HTML5
- **Styling**: CSS3 (Grid, Flexbox, Custom Properties, Media Queries)
- **Design System**: Microsoft Fluent Design System
- **Fonts**: Segoe UI (system default), IBM Plex Mono (monospace)
- **Data Source**: Nightscout REST API
- **Launch**: VS Code F5 debugger with Edge/Chrome
- **No build step required** – works directly in modern browsers

### Future Stack (Planned)
- **Build Tool**: Vite
- **Language**: TypeScript
- **3D Engine**: Three.js or Babylon.js
- **Component Library**: React Three Fiber (optional)
- **Data Visualization**: D3.js for advanced transformations
- **Testing**: Vitest + Playwright

## Design Principles

### Microsoft Fluent Design System
This project strictly follows Fluent Design guidelines:

1. **Clean & Minimal**: Content-first approach with ample whitespace
2. **Acrylic/Glass Effects**: Subtle depth using transparency and backdrop-filter
3. **Hierarchy**: Clear visual distinction between primary, secondary, and tertiary content
4. **Responsive**: Mobile-first design that scales elegantly to desktop
5. **Accessible**: WCAG AA compliance, keyboard navigation, screen reader support
6. **Smooth Animations**: Purposeful transitions (140ms ease by default)
7. **Dark Mode**: Full theme support with CSS variables

### Color Palette
- **Light Theme**:
  - Background: `#f4fff8` to `#f6f9ff` (subtle gradient)
  - Surface: `rgba(255, 255, 255, 0.88)` (acrylic)
  - Text: `#13212c` (high contrast)
  - Accent: `#0b8f8f` (teal)

- **Dark Theme**:
  - Background: `#0a1618` to `#101828` (dark gradient)
  - Surface: `rgba(16, 30, 40, 0.84)` (acrylic)
  - Text: `#e8f1f8` (high contrast)
  - Accent: `#24c6c0` (bright teal)

- **Status Colors**:
  - Low glucose (`< 70 mg/dL`): `#f59e0b` (amber)
  - Target (`70–180 mg/dL`): `#0ea86b` (green)
  - High glucose (`> 180 mg/dL`): `#dc2626` (red)

## Key Features & Implementation

### 1. 24-Hour Glucose Dashboard
- **Latest Value Display**: Large hero text showing current glucose with trend direction
- **Sparkline Chart**: SVG-based 24h trend visualization
- **Summary Metrics**: Average, min, max, data point count
- **Time-in-Range**: Stacked bar showing % low/target/high
- **Real-time Unit Conversion**: mg/dL ↔ mmol/L toggle
- **Theme Toggle**: Light/Dark mode switch with CSS variable swapping

### 2. Token Security & Validation
- **Safe Scope Detection**: Probes DELETE endpoint on non-existent record to detect write permissions
- **Error Handling**: Clearly indicates if token is not reader-only
- **No Data Modification**: Only performs read operations and safe probes
- **URL Parsing**: Extracts base URL and token from pasted URL

### 3. Responsive Design
- **Desktop**: 2-column layout (hero panel + metrics card)
- **Tablet**: Single column with stacked elements
- **Mobile**: Full-width, optimized touch targets

## Code Style Guidelines

### HTML
- Semantic elements (`<article>`, `<aside>`, `<main>`, etc.)
- ARIA labels for accessibility
- Avoid `<div>` soup – use descriptive class names
- Form validation built-in (no extra libraries)

### CSS
- Use CSS custom properties (variables) for theming
- Prefer CSS Grid for layout, Flexbox for alignment
- No inline styles – keep all CSS in `<style>` tag
- Vendor prefixes not needed for modern browsers
- Mobile-first media queries

### JavaScript
- Vanilla ES6+ (no npm dependencies required)
- Async/await for API calls, not `.then()`
- Clear variable names – avoid single-letter except in loops
- Comments for complex logic
- No logging – use browser DevTools

## Common Tasks

### Adding a New UI Element
1. Add HTML structure to `.panel` or `.dashboard`
2. Define CSS variables for colors in `:root` and `body[data-theme="dark"]`
3. Style with Flexbox/Grid, respecting 8px spacing grid
4. Wire up JavaScript listeners for interactivity
5. Test in both light and dark modes
6. Verify touch targets are >= 44px

### Fixing a Bug
1. Check browser console for errors (F12)
2. Inspect HTML/CSS via DevTools
3. Use debugger breakpoints in JS
4. Test fix in both themes and at multiple viewport sizes
5. Verify no regression in other features

### Preparing for 3D Phase
1. Create `/src` directory structure
2. Set up TypeScript config + Vite
3. Create Nightscout API client (`api/nightscout.ts`)
4. Implement data transformation pipeline (`data/transform.ts`)
5. Scaffold Three.js scene setup (`scene/setup.ts`)
6. Port current UI to new build process

## Contribution Workflow

When asked to modify or add features:
1. **Understand the current state** – read relevant files
2. **Respect the design system** – use Fluent principles and existing colors
3. **Maintain compatibility** – no breaking changes without discussion
4. **Test thoroughly** – verify in all themes and viewports
5. **Document changes** – update README if needed
6. **Keep it simple** – prefer vanilla JS over frameworks
